¿Qué es Zymos?

Zymos es una plataforma de gestión integral diseñada exclusivamente para cervecerías artesanales. Centraliza en un solo lugar todos los procesos del negocio: producción, inventario, equipos, inocuidad alimentaria, ventas, facturación y análisis.

Las cervecerías artesanales gestionan simultáneamente recetas técnicas, lotes en distintas fases de fermentación, insumos perecederos, equipos con mantenimiento, clientes y proveedores — frecuentemente con hojas de cálculo y herramientas dispersas que no se comunican entre sí. Zymos resuelve este problema con una plataforma unificada, accesible desde cualquier navegador sin instalación de software.

Como plataforma SaaS multi-tenant, cada cervecería opera en su propio entorno aislado con su marca, colores y datos privados — mientras comparte la misma infraestructura robusta actualizada automáticamente.

Seguridad y Control de Acceso

Zymos implementa un modelo de seguridad en capas que cubre autenticación, autorización granular, aislamiento de datos y cumplimiento de inocuidad alimentaria:

🏷️
RBAC dinámico por tenant Cada cervecería define sus propios roles con permisos granulares por módulo (Ver / Crear / Editar / Eliminar). 5 roles de sistema + roles custom ilimitados.
🔑
Autenticación robusta Contraseñas BCrypt, bloqueo automático por intentos fallidos configurable (intentos y duración en minutos).
🛡️
CSP enforced Content Security Policy activa. Sin scripts inline. Nonces dinámicos en todos los bloques de código de cada página.
🍪
Sesiones seguras Timeout de 30 min. Cookies HttpOnly + Secure + SameSite=Strict en producción. Sesión inválida → redirect a login.
🏢
Aislamiento multi-tenant Filtro @TenantId de Hibernate en todas las entidades. Imposible acceder a datos de otro tenant incluso con queries directas o tokens válidos.
🏥
Control de salud BPM El sistema verifica el registro diario de salud de cada operario y bloquea el acceso si hay síntomas pendientes de autorización por un administrador.
⚡
API JWT + Rate Limiting Tokens HS256 con TTL configurable. Límite de 100 req/min por IP con respuesta HTTP 429. SpringDoc OpenAPI para documentación.
📝
Auditoría de accesos Cada login registrado en base de datos en transacción independiente — persiste aunque la sesión falle o se revierta.

Arquitectura SaaS Multi-Tenant

Zymos opera como una plataforma compartida donde cada cervecería es un tenant independiente con total aislamiento de datos:

🌐

Subdominio propio

Cada cliente accede por
micerveceria.zymos.app

🔒

Datos aislados

Filtro @TenantId en todas las entidades — sin mezcla posible

🎨

Branding propio

Logo, colores y tipografías configurables por cliente

⚡

Actualizaciones

Mejoras desplegadas para todos los clientes simultáneamente

Esta arquitectura permite escalar el número de clientes sin incremento proporcional de costos operativos, manteniendo la seguridad y el rendimiento para todos.

Diferenciadores Clave

1
Multi-tenant real — sin silos de instancias Un solo deploy sirve a N cervecerías con aislamiento completo de datos, branding propio y roles independientes por cliente. Sin multiplicar infraestructura por cada cliente nuevo.
2
BPM embebido en el flujo operativo diario El módulo de inocuidad alimentaria no es un addon externo. El sistema bloquea activamente el acceso de operarios con síntomas de salud pendientes — con firma digital del administrador para autorizar la excepción.
3
RBAC granular configurado por la cervecería Los roles no son fijos del sistema. Cada cliente los crea, nombra y configura con permisos por módulo (Ver / Crear / Editar / Eliminar). Un operario solo ve exactamente lo que necesita.
4
Onboarding acelerado desde datos existentes El migrador XLSX permite importar el historial completo — recetas, lotes, proveedores, facturas, clientes, equipos — en minutos desde las hojas de cálculo actuales de la cervecería.
5
Branding 100% personalizable por cliente Logo, colores primario, acento y navbar, tipografías — todo configurable desde el panel de administración sin reiniciar la aplicación. La plataforma se ve como si fuera desarrollada exclusivamente para la marca del cliente.
